Part 1: Artificial Intelligence Landscape

What is Artificial Intelligence?

Artificial Intelligence (AI) is the simulation of human intelligence processes by machines, especially computer systems.

Core Capabilities

  • 🧠 Reasoning — logical problem solving
  • 📚 Learning — improving from experience
  • 👁️ Perception — interpreting sensory input
  • 💬 Natural Language — understanding text/speech
  • 🤖 Planning — goal-directed action

Application Domains

  • Drug discovery & design
  • Medical image analysis
  • Clinical decision support
  • Genomics & proteomics
  • Pharmacovigilance

The AI Hierarchy

AI: Artificial Intelligence

the science and engineering of making intelligent machines, especially intelligent computer programs

ML: Machine Learning

design and train algorithms to learn from and act on data

DL: Deep Learning

multi-layered neural networks, called deep neural networks, to simulate the complex decision-making power of the human brain

GenAI: Generative Deep Learning

a subset of AI used to produce text, images, videos, or others response to specific prompts. 1

Part 2: Machine Learning

What is Machine Learning?

Machine Learning (ML) is a field of AI that enables systems to learn and improve from experience without being explicitly programmed.

Instead of writing rules:

IF temperature > 38.5 THEN fever = True
IF fever AND cough THEN risk = High
...

We let the algorithm discover patterns from labeled examples.

Machine Learning Key Components

  1. Data — examples & labels
  2. Model — mathematical function
  3. Loss function — how wrong are we?
  4. Optimization — minimize loss

Types of Machine Learning

Supervised Learning

  • Labeled training data
  • Learn input → output mapping
  • Examples:
    • Classification
    • Regression
  • Applications: diagnosis, prognosis

Unsupervised Learning

  • No labels required
  • Find hidden structure
  • Examples:
    • Clustering
    • Dimensionality reduction
  • Applications: patient stratification

Reinforcement Learning

  • Agent + environment
  • Learn by reward/penalty
  • Examples:
    • Game playing
    • Drug dosing optimization
  • Applications: clinical decision

Supervised Learning in Detail

Classification — predict discrete categories (e.g., malignant vs. benign)

Regression — predict continuous values (e.g., drug concentration, IC50)

Model Properties

  • Performance
  • Linearity
  • Interpretability
  • Scalability
  • Robustness

How Machine Learning Works

Key Concepts: Bias-Variance Tradeoff

  • Underfitting (High Bias)
    • Model too simple
    • Poor on training AND test data
    • Fix: more complex model, more features
  • Overfitting (High Variance)
    • Model too complex
    • Great on training, poor on test
    • Fix: regularization, more data, simpler model
  • Sweet Spot
    • Balances bias & variance
    • Generalizes well to new data

📉 Train Error (solid) decreases with complexity
📈 Test Error (line) — U-shaped — optimal at middle
⬆️ Left = High Bias (Underfitting)
⬆️ Right = High Variance (Overfitting)

Part 3: Deep Learning

What is Deep Learning?

Deep Learning (DL) uses artificial neural networks with many layers to learn hierarchical representations from raw data.

Why “Deep”?

  • Multiple hidden layers
  • Each layer learns more abstract features
  • Pixels → Edges → Shapes → Objects

Key Advantage

  • Automatic feature extraction — no manual feature engineering needed
  • Excels at images, text, audio, sequences
Input Layer   Hidden Layers   Output Layer
   [x₁] ──┐
   [x₂] ──┤─→[●][●][●]─→[●][●]─→[ŷ]
   [x₃] ──┤─→[●][●][●]─→[●][●]
   [x₄] ──┘
   
   Each ● = neuron (weighted sum + activation)

Neural Network Components

Neuron (Perceptron): z = \sum_{i} w_i x_i + b, \quad \hat{y} = \sigma(z)

Activation Functions:

Function Formula Use Case
Sigmoid 1/(1+e⁻ᶻ) Binary output
ReLU max(0, z) Hidden layers (CNN)
Softmax eᶻⁱ/Σeᶻʲ Multi-class output
Tanh (eᶻ-e⁻ᶻ)/(eᶻ+e⁻ᶻ) RNN hidden states

Training: Backpropagation + Gradient Descent

w \leftarrow w - \eta \cdot \frac{\partial \mathcal{L}}{\partial w}

Deep Learning Architectures

Architecture Key Feature Applications
Feedforward (FNN/MLP) Fully connected layers Tabular data classification

Recurrent (RNN/LSTM) Sequential/temporal data EHR time series, text

The Titanic Dataset

Kaggle Titanic Dataset

Objective: Predict whether a passenger survived the Titanic disaster.

This is a Binary Classification Problem: - Target: Survived (0 = No, 1 = Yes)

Feature Type Description
Pclass Ordinal Passenger class (1, 2, 3)
Sex Categorical male / female
Age Continuous Age in years
SibSp Integer # siblings/spouses aboard
Parch Integer # parents/children aboard
Fare Continuous Ticket fare
Embarked Categorical Port (C, Q, S)

Step 3: Data Preprocessing

using DataFrames, CategoricalArrays, Statistics

df_clean = copy(df)

# 1. Drop irrelevant columns
select!(df_clean, Not([:PassengerId, :Name, :Ticket, :Cabin]))

# 2. Impute missing Age with median
age_median = median(skipmissing(df_clean.Age))
df_clean.Age = coalesce.(df_clean.Age, age_median)

# 3. Impute missing Embarked with mode
df_clean.Embarked = coalesce.(df_clean.Embarked, "S")

# 4. Encode Sex: female=1, male=0
df_clean.Sex = ifelse.(df_clean.Sex .== "female", 1, 0)

# 5. One-hot encode Embarked
df_clean.Embarked_C = Int.(df_clean.Embarked .== "C")
df_clean.Embarked_Q = Int.(df_clean.Embarked .== "Q")
select!(df_clean, Not(:Embarked))

# 6. Feature engineering
df_clean.FamilySize = df_clean.SibSp .+ df_clean.Parch .+ 1
df_clean.IsAlone = Int.(df_clean.FamilySize .== 1)

println("Processed shape: ", size(df_clean))

Step 4: Prepare MLJ-Compatible Data

using MLJ

# Separate features and target
target = :Survived
features = setdiff(names(df_clean), [string(target)])

X = select(df_clean, features)
y = df_clean[:, target]

# Convert target to categorical (required for classification)
y = CategoricalArray(y)

# Coerce feature types for MLJ
X = coerce(X, 
    :Pclass     => OrderedFactor,
    :Sex        => Count,
    :Age        => Continuous,
    :SibSp      => Count,
    :Parch      => Count,
    :Fare       => Continuous,
    :FamilySize => Count,
    :IsAlone    => Count
)

println("X:", schema(X))
println("y:", scitype(y))

Step 6: Train a Decision Tree

using MLJ
MLJ.@load DecisionTreeClassifier pkg=DecisionTree

Step 7: Train a Random Forest

MLJ.@load RandomForestClassifier pkg=DecisionTree

# Define model with hyperparameters

Step 8: Cross-Validation Evaluation

using MLJ

# 5-fold cross-validation
cv = CV(nfolds=5, shuffle=true, rng=42)

# Evaluate multiple metrics at once
eval_rf = evaluate(rf_clf, X, y,
    resampling = cv,
    measures   = [accuracy, f1score, auc],
    verbosity  = 0
)

println("\n=== Random Forest Cross-Validation ===")
for (m, v) in zip(eval_rf.measure, eval_rf.measurement)
    println("$(m):  ", round(v, digits=4))
end
println("Per-fold accuracy: ", round.(eval_rf.per_fold[1], digits=3))

Step 9: Hyperparameter Tuning

using MLJ

# Define search range
r1 = range(rf_clf, :n_trees,      lower=50,  upper=300, scale=:linear)
r2 = range(rf_clf, :max_depth,    lower=3,   upper=15)
r3 = range(rf_clf, :n_subfeatures, lower=2,  upper=6)

# Grid search with cross-validation
tuned_rf = TunedModel(
    model     = rf_clf,
    resampling = CV(nfolds=5, rng=42),
    tuning    = Grid(resolution=5),
    range     = [r1, r2, r3],
    measure   = accuracy
)

mach_tuned = machine(tuned_rf, X, y)
fit!(mach_tuned, rows=train, verbosity=0)

# Best model
best_model = fitted_params(mach_tuned).best_model
println("Best parameters:")
println("  n_trees:       ", best_model.n_trees)
println("  max_depth:     ", best_model.max_depth)
println("  n_subfeatures: ", best_model.n_subfeatures)

Step 11: Model Comparison

using MLJ

# Load all model types
MLJ.@load LogisticClassifier pkg=MLJLinearModels
MLJ.@load XGBoostClassifier  pkg=XGBoost

models = [
    ("Logistic Regression", LogisticClassifier()),
    ("Decision Tree",       DecisionTreeClassifier(max_depth=5)),
    ("Random Forest",       RandomForestClassifier(n_trees=100)),
    ("XGBoost",             XGBoostClassifier(num_round=100)),
]

results = []
for (name, model) in models
    ev = evaluate(model, X, y,
        resampling = CV(nfolds=5, rng=42),
        measures   = [accuracy, f1score],
        verbosity  = 0)
    push!(results, (
        Model    = name,
        Accuracy = round(ev.measurement[1], digits=4),
        F1_Score = round(ev.measurement[2], digits=4)
    ))
end

println(DataFrame(results))

Summary

AI / ML / DL Concepts

  • AI is the broad field of intelligent machines
  • ML learns patterns from data automatically
  • DL uses deep neural networks for complex tasks
  • Bias-variance tradeoff is fundamental
  • Always evaluate with cross-validation

  1. Collect & explore data (EDA)
  2. Preprocess (clean, encode, engineer features)
  3. Split: train / validation / test
  4. Train & tune models
  5. Evaluate & compare
  6. Deploy & monitor
